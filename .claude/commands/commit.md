ステージ済みの変更に対してコミットメッセージを作成し、ユーザーの承認後にコミットを実行してください。

手順:
1. `git status` でステージ済みファイルを確認する
2. `git diff --cached` でステージ済みの差分を確認する
3. `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
4. 変更内容を分析し、日本語でコミットメッセージを作成する
5. ユーザーに提案して承認を得る
6. 承認後にコミットを実行する

コミットメッセージのルール:
- 1行目: 変更の要約（日本語）
- 空行
- 本文: 変更の詳細を箇条書き（必要な場合のみ）
- Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com> を末尾に付与する

注意:
- ステージ済みの変更がない場合はその旨を伝えて終了する
- ユーザーの承認なしにコミットを実行しない
